Object Oriented Programming (OOP) tidak hanya sekadar teknik pemrograman, tetapi merupakan pendekatan desain sistem yang meniru cara manusia memahami dunia nyata. Dalam kehidupan sehari-hari, kita mengenali entitas sebagai objek yang memiliki karakteristik dan perilaku.

OOP bertujuan untuk meningkatkan modularitas sistem dengan memecah program besar menjadi bagian-bagian kecil dalam bentuk class sehingga lebih mudah dikembangkan dan dipelihara.

Selain itu OOP mendukung prinsip reusability yaitu penggunaan ulang kode sehingga developer tidak perlu menulis ulang logika yang sama.

Empat pilar utama OOP adalah Encapsulation, Abstraction, Inheritance, dan Polymorphism.

Paradigma ini menjadi fondasi penting dalam pengembangan aplikasi modern termasuk framework Flutter yang menggunakan bahasa Dart.

1. Class & Object (Car)

Class merupakan blueprint dari objek. Sedangkan object adalah implementasi nyata dari class tersebut.

2. Constructor

Constructor merupakan metode khusus yang dipanggil secara otomatis saat objek dibuat. Constructor digunakan untuk menginisialisasi nilai awal dari atribut objek.

3. Validasi Constructor (Mahasiswa)

Validasi constructor digunakan untuk memastikan data yang dimasukkan ke dalam objek berada pada kondisi yang valid.

4. Encapsulation (Person)

Encapsulation adalah teknik menyembunyikan data internal objek agar tidak dapat diakses secara langsung dari luar class.

5. Named Parameter (Produk)

Named parameter memungkinkan pengisian parameter berdasarkan nama variabel sehingga kode menjadi lebih jelas dan mudah dipahami.

Kesimpulan

Object Oriented Programming merupakan paradigma pemrograman yang memodelkan sistem menggunakan objek yang memiliki data dan perilaku. Dengan menggunakan konsep seperti class, object, constructor, validasi dan encapsulation, program dapat dibuat lebih terstruktur dan mudah dipelihara.

Dalam bahasa Dart, konsep OOP dapat diterapkan dengan sangat baik sehingga mempermudah developer dalam membangun aplikasi yang scalable, maintainable dan robust.
